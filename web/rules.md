# Wanxiang Agent 前端规范

本规范适用于 `web/` 下的全部前端代码、样式、状态、接口调用和测试。用户的最新明确要求优先于本文。

## 1. 目录与职责

- `src/api/`：统一封装 HTTP 请求、鉴权和接口类型。
- `src/components/`：可复用组件，不直接复制页面级业务逻辑。
- `src/stores/`：Pinia 状态及持久化边界。
- `src/views/`：路由页面，只保留页面自身内容，不重复实现全局导航、鉴权或布局。
- `test/`：唯一允许保存测试文件的目录，按 `api/`、`components/`、`stores/`、`views/` 镜像源码领域。
- `src/` 中禁止出现 `*.test.*`、`*.spec.*` 或临时验证脚本。

## 2. 组件与交互

- 优先使用项目已经安装的 Element Plus 组件，不重复编写已有的菜单、布局、表单、弹窗和反馈组件。
- 全局导航统一由 `AdminShell.vue` 管理；业务页面不得再次创建顶部栏或侧栏。
- 导航必须支持桌面展开、桌面折叠和移动端抽屉。展开时显示图标和文字，折叠时图标必须统一居中。
- 路由、导航和 Tab 必须使用同一份路由元数据，不维护互相独立的菜单列表。
- 所有按钮必须有可识别文字或 `aria-label`；只显示图标的按钮必须提供可访问名称。
- 提交、删除、发布、回滚等操作必须有明确的 loading、防重复提交和高风险确认状态。

## 3. Tab 与本地状态

- 首次进入后台且没有 Tab 时显示 Dashboard 内容。
- 点击导航打开唯一 Tab；重复点击只激活已有 Tab。
- 所有 Tab 可关闭；关闭到零个时回到无标签 Dashboard。
- 需要持久化的状态必须使用带版本号的 localStorage key，结构变化时升级版本，不兼容读取旧结构。
- 本地状态只能保存界面恢复所需字段，不保存密钥、访问令牌、用户隐私或完整接口响应。

## 4. API、鉴权与 SSE

- 所有普通接口必须通过 `src/api/client.ts` 调用，不允许页面各自拼装鉴权请求。
- 管理接口收到 401 时必须清理过期登录态和工作区状态，并跳转登录页，同时保留原页面 redirect。
- 登录和初始化接口的 401 只展示凭据错误，不触发重复跳转。
- SSE 使用同源 HttpOnly Cookie 鉴权，不把 Token 放入 URL、日志或界面。
- API 错误必须进入页面可见状态或统一登录失效流程，禁止只写控制台后让页面保持无响应。

## 5. 响应式与样式

- 桌面、平板和移动端必须共享同一业务组件，使用响应式布局调整展示方式。
- 新样式优先使用现有 CSS 变量；禁止无理由新增相近颜色、阴影或间距常量。
- 覆盖 Element Plus 内部样式时优先给受控节点绑定明确业务类，避免依赖不稳定的深层 DOM 结构。
- 横向内容必须设置 `min-width: 0` 或明确滚动容器，不能撑破后台主内容区。
- 动画必须简短，并尊重 `prefers-reduced-motion`。

## 6. 测试文件规范

- 所有测试只能放在 `web/test/`，禁止与 `src/` 源码混放。
- 测试目录按源码职责分层，例如：
  - `src/api/client.ts` 对应 `test/api/client.test.ts`
  - `src/components/AdminShell.vue` 对应 `test/components/AdminShell.test.ts`
  - `src/stores/tasks.ts` 对应 `test/stores/tasks.test.ts`
- `npm test` 只运行 `web/test/` 下的测试。
- 为当前任务新增的测试默认视为临时验证文件；功能验证和生产构建完成后，必须删除对应新增测试文件，禁止把临时测试遗留在仓库造成污染。
- 只有用户明确要求长期保留，或现有基线测试需要迁移维护时，测试文件才可以保留；保留的测试仍必须位于 `web/test/`。
- 删除临时测试前必须先记录其执行命令和通过结果；删除后再次运行现有测试与生产构建，确认源码不依赖临时测试。
- 测试不得写入真实密钥、生产 Token、用户密码或不可逆生产数据。

## 7. 验证与部署

- 涉及 `web/src/`、依赖、Vite 或前端运行配置时，必须执行：

```bash
cd web
npm test -- --run
npm run build
```

- `npm test`、TypeScript 检查或生产构建任一失败时不得部署。
- 生产静态目录为 `web/dist`；构建后必须确认 `index.html` 引用新资源哈希。
- 仅前端修改不重启 `wanxiang-agent`；后端/API 契约修改才按 PM2 规则重启。
- 部署后必须验证页面入口、关键静态资源和 `/api/health`，并确认工作区没有误提交临时测试或构建产物。
