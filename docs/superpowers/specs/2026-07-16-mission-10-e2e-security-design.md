# M10 端到端验收与安全加固设计

## 目标

用可重复的自动化场景证明 M01-M09 链路，而不是新增第二套调度实现。测试使用临时根目录、SQLite、临时 Git 仓库和假 Provider；不得使用真实密钥或触发真实生产发布。

## 场景

端到端夹具覆盖单 Agent 全链路、多 Agent 负责人和依赖顺序、缺配置恢复、Worker 中断后原 Agent 恢复、新 Agent 接管、阻塞 Issue、M09 测试/构建/发布确认与回滚。安全矩阵覆盖过期租约、越界路径、无 scope Token、伪造身份、符号链接、命令注入和敏感信息。

## 证据

每个场景断言数据库状态、runtime event 时间线、Git 提交/分支、API 状态码和密钥扫描。M10 只在全部后端、前端和端到端测试通过后完成，并同步 Mission 的真实证据。生产部署仍按 `wanxiangAgent.md` 第 17 节单独确认、构建、备份、PM2 重启和健康检查。
